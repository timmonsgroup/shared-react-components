## PAM Layout High Level Structure

At the top level the PAM layout contains the following properties:

<table>
    <tr>
        <td>Name</td>
        <td>Required</td>
        <td>Type</td>
        <td>Description</td>
    </tr>
    <tr>
        <td>id</td>
        <td>X</td>
        <td>int</td>
        <td>The id of the layout</td>
    </tr>
    <tr>
        <td>name</td>
        <td>X</td>
        <td>string</td>
        <td>The name of the layout</td>
    </tr>
    <tr>
        <td>data</td>
        <td>X</td>
        <td>Layout Data Options</td>
        <td>A data options configuration</td>
    </tr>
    <tr>
        <td>grid</td>
        <td></td>
        <td>Grid Data Options</td>
        <td>A grid options configuration</td>
    </tr>
    <tr>
        <td>modelID</td>
        <td>X</td>
        <td>int</td>
        <td>ID of the model this layout belongs to</td>
    </tr>
    <tr>
        <td>enabled</td>
        <td>X</td>
        <td>boolean</td>
        <td>Is this layout enabled?</td>
    </tr>
    <tr>
        <td>editable</td>
        <td>X</td>
        <td>boolean</td>
        <td>Can data be manipulated from this layout?</td>
    </tr>
    <tr>
        <td>layoutKey</td>
        <td>X</td>
        <td>String</td>
        <td>Key of the layout</td>
    </tr>
    <tr>
        <td>type</td>
        <td>X</td>
        <td>int</td>
        <td>Layout type. 1 for form, 2 for grid.</td>
    </tr>
    <tr>
        <td>sections</td>
        <td>X</td>
        <td>List of Grid Layout Section</td>
        <td>A list of sections in this layout</td>
    </tr>
</table>


Example

```json
{
    "id": 12,
    "name": "Test Layout",
    "data": {},
    "grid": {},
    "modelId": 30,
    "enabled": true,
    "editable": false,
    "layoutKey": "list",
    "type": 2,
    "sections": []
}
```
